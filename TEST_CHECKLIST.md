# 테스트 체크리스트

## UI/UX 테스트

### ✅ 디자인
- [ ] 보라색 그라데이션 제거 확인 (중립적인 배경 사용)
- [ ] 포인트 컬러 1-2개만 사용 (파란색 primary)
- [ ] 카테고리 칩 스타일 확인 (필터 칩 형태)
- [ ] 뉴스 카드 통일성 확인 (썸네일/제목/요약/출처/시간)
- [ ] 새로고침 버튼 위치 확인 (상단 우측)

### ✅ 반응형
- [ ] 모바일 (320px~): 1열 그리드 확인
- [ ] 태블릿 (640px~): 2열 그리드 확인
- [ ] 데스크톱 (1024px~): 3열 그리드 확인
- [ ] 터치 영역 최소 44px 확인

### ✅ 접근성
- [ ] 키보드 네비게이션 작동 (Tab, Enter)
- [ ] 포커스 표시 명확함
- [ ] 대비 비율 충분함
- [ ] 스크린 리더 호환성

### ✅ 상태 UI
- [ ] 로딩 상태: 스피너 + 텍스트 표시
- [ ] 에러 상태: 아이콘 + 메시지 + 재시도 버튼
- [ ] 빈 상태: 아이콘 + 안내 메시지
- [ ] 토스트 알림 작동

## 크롤링 테스트

### ✅ 안정성
- [ ] 각 카테고리에서 최소 10개 이상 수집
- [ ] 타임아웃/차단 시 재시도 작동
- [ ] 에러 발생 시에도 빈 배열 반환 (앱 크래시 없음)

### ✅ 중복 제거
- [ ] URL 정규화로 중복 제거 확인
- [ ] 제목 유사도로 중복 제거 확인 (85% 이상)
- [ ] 동일 기사 중복 없음

### ✅ 데이터 품질
- [ ] 이미지: og:image 우선 추출 확인
- [ ] 요약: meta description 우선, 본문 2-3문장 fallback
- [ ] 요약 길이: 80-160자 범위 확인
- [ ] 날짜: published_time 우선, time 태그 fallback
- [ ] 날짜 형식: 상대 시간 표시 (N분 전, N시간 전)

### ✅ 정렬
- [ ] 최신순 정렬 확인 (발행시간 기준)
- [ ] 발행시간 없으면 수집시간 기준

## API 테스트

### ✅ 엔드포인트
```bash
# 카테고리 목록
curl http://localhost:8000/api/categories

# 뉴스 조회
curl "http://localhost:8000/api/news?category=health"

# 새로고침
curl "http://localhost:8000/api/news/refresh?category=health"

# 헬스 체크
curl http://localhost:8000/health
```

### ✅ 응답 형식
- [ ] 성공 시: `{success: true, articles: [...], count: N}`
- [ ] 실패 시: `{success: false, message: "...", articles: []}`
- [ ] 모든 기사에 필수 필드 포함: id, title, url, source, publishedAt, imageUrl, summary

### ✅ 캐싱
- [ ] 첫 요청 시 크롤링 수행
- [ ] 5분 내 재요청 시 캐시 반환
- [ ] 새로고침 시 캐시 무효화

## 성능 테스트

- [ ] 페이지 로드 시간 < 2초
- [ ] 카테고리 선택 후 뉴스 표시 < 3초
- [ ] 이미지 로딩 실패 시 플레이스홀더 표시
- [ ] 동시 요청 처리 (5-10개)
